# Learning Journal

Short weekly notes. Not a diary — a working log you'll mine later for portfolio material, blog posts, and talk submissions.

## What to Track

Each week, commit one markdown file: `week-NN.md`. Three sections, three to five sentences each. Don't go longer.

### 1. What worked
Specific patterns, prompts, agents, or workflows that produced unexpectedly good results. "Worked" means the output exceeded your expectations or saved meaningful time, not that it merely ran.

### 2. What didn't
Specific failures: a prompt that consistently misfired, a subagent whose description didn't trigger reliably, an MCP server that fought you, a methodology framework that felt heavy for the task. Be specific enough that future-you can avoid the trap.

### 3. What I'd do differently
The forward-looking version. Sometimes it's a small habit ("plan in CLI, execute in Desktop"); sometimes it's a structural choice ("vertical slice would have shipped feature X two days sooner"). Don't write generic resolutions — write the specific decision rule.

## What Not to Bother With

- Detailed code logs. Git history exists.
- Daily emotional check-ins. This isn't therapy.
- Tool inventories. Keep those in `CLAUDE.md` or the main repo.
- Anything you wouldn't be willing to publish in a blog post.

## Why This Becomes Portfolio Material

After Week 24 you'll have ~24 entries. The pattern of "tried X → got Y → adjusted to Z" across six months is exactly what makes a credible "I spent six months learning Claude Code" blog post or conference talk. Generic "AI is cool" content is everywhere; specific lessons-learned with timestamps and code commits are not.

When you write your launch post for the capstone, the journal is where you mine for narrative. When you pitch a talk submission, the journal is where you find the through-line.

## Suggested File Format

```markdown
# Week NN — Phase X

## What worked
- ...
- ...

## What didn't
- ...

## What I'd do differently
- ...

## Links
- [PR #...]
- [Commit ...]
- [Tweet, blog post, etc.]
```

## Phase-End Entries

After each phase, in addition to weekly entries, write one longer `phase-N-retrospective.md` (500–800 words) covering:
- The biggest mental model shift this phase
- The decision you'd make differently if you redid the phase
- One specific pattern you're carrying forward
- One thing you tried that you'd not recommend to someone else

These are your strongest portfolio pieces.

## Daily Entries (Optional)

Some phases — especially Week 1, Week 14, and capstone weeks — benefit from daily entries. The Week 1 walkthrough in [Appendix A](../appendix-a-week-1-walkthrough.md) suggests three. Don't try to keep daily entries through the whole 24 weeks; you'll quit by Week 6.
